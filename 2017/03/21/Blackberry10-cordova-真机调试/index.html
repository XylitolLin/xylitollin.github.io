Blackberry10 cordova 真机调试

发表于 2017-03-21

前言

刚好入手了一台 blackberry passport，作为一位什么都折腾一番的男人，我觉得为这台手机开发几个 APP。因为主职是前端，cordova 在 iOS 和 android 也开发过。所以选用 cordova 为开发平台了。

环境准备(windows 7)

  1. 安装 cordova 官方安装教程
  2. 安装 blackberry10 WebWorks 2.2 官方安装教程
  3. 安装 blackberry link 官方下载地址

考虑到 WebWorks 已经很久没有人维护了，自带的 cordova 也比较旧。之后我们的开发是不会通过 WebWorks 进行的。安装 WebWorks 主要是为了 bb-tools 这个文件夹里面的开发工具，这是进行开发和真机调试必备工具。

这里需要把 bb-tools 这个文件夹的路径添加到系统的环境变量 PATH 才能方便我们使用。

⚠️WebWorks 需要依赖 java 6

真机调试

由于篇幅问题，这次重点记录真机调试的准备工作和很多官方文档没有提及的坑。上述的所需软件安装教程不赘述。

获取 signing key

进入 BlackBerry Keys Order Form

按需选择，这里我们选择第一个。然后经过一系列 next 、填写密码的操作，最后我们会得到一个 csk 文件。有效期为一年，并且告诉我们要放到对应的文件夹。那么，没有安装 native sdk 的我们其实是没有这几个文件夹的，我们也不需要，保存在任意文件就可以了。


生成 debug token (*.bar)

  1. 在 cmd 中进入保存了 csk 的文件夹。
  2. 执行 blackberry-signer -csksetup -cskpass (your password) 这里的 password 是在上一步获取 signing key 填的 password。
  3. 执行 blackberry-keytool -genkeypair -keystore (output file name) -storepass <password> -author (Your Name) 成功后，我们会获得一个 .p12 的文件。

  4. 执行 blackberry-debugtokenrequest -cskpass (your password) -keystore (your keystore file) -storepass (another password) -deviceId 0x(your device PIN) (output file name) 成功后，会生成一个 .bar 的文件，这个就是我们需要的 debug token。

将 debug token 文件导入手机

⚠️确保你的手机开启了开发模式

回到保存有 debug token 文件的文件夹下，执行:
blackberry-deploy -installDebugToken (your debug token) -device (IP address) -password (device password)
成功后，会在手机的开发调试的页面显示调试令牌的相关信息。

首次开发

按照 官方教程 一步步进行即可。

这里我们重点关注执行了 cordova run blackberry10 之后的问题：

  1. Q：blackberry-nativepackager not found

    A: 将 bb-tools 这个文件件的路径添加到系统环境变量

  2. Q：failure 881 application author does not match debug token author

    A: 将 debug token 复制一份保存并改名 C:\Users.cordova\blackberry10debugtoken.bar
