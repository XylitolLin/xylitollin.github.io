如果有朋友说不理解为什么 setLensPostion 这个函数的原理，可以看上图加以理解。因为 mousemove 事件获得的坐标是相对 document 的。所以，在 componentDidMount 我们先获取到 mousemove 区域相对 document 的坐标和长宽。e.clientX - dom.left 就是我们鼠标在图片区域滑动的 X 轴距离。最后，我们需要小黄块的中心随鼠标而动，所以小黄块实际需要改变的 left 值是：鼠标滑动距离 － 小黄块宽度 / 2。

zoom window

这里先理清一下我们要实现的效果，大图窗口显示的是 lens 在小图对应的内容，不多不少。所以 zoom window 其实就是相当于大图的 lens。我的语文不及格，不知道怎么阐述这个关系，反正得出结论就是：小黄块 ( lens ) 的宽高是和小图的宽高、大图的宽高、zoom window 的宽高有关系的，

预告

上面，我们基本实现了 image zoom 的功能，体验和京东的基本一致。但是还是有几个地方值得去优化的：

  1. 图片的宽高应该自动获取。
  2. 图片的加载应该优化。
  3. 有 React 组件开发经验的朋友应该看出了，其实我们可以把上面的组件写成 stateless component 这样代码就可以更简洁了。
